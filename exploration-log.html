Date: March 8, 2026

Q&A Session

What is this project? Give me a high-level summary of what Excalidraw does and what technologies it uses.

Excalidraw is an open-source virtual whiteboard for creating hand-drawn style diagrams. It features an infinite canvas, real-time collaboration with end-to-end encryption, export to PNG/SVG/JSON, dark mode, and PWA offline support. Built with React 19, TypeScript, Vite, and Vitest, managed as a Yarn monorepo.

What is the folder structure of this project?

A monorepo with these key folders:

  • packages/excalidraw/ — core npm library
  • packages/element/, packages/math/, packages/common/, packages/utils/ — supporting packages
  • excalidraw-app/ — the full excalidraw.com web app
  • examples/ — Next.js and browser integration examples
  • firebase-project/ — collab backend config
  • scripts/ — build/release automation
  • .github/ — CI/CD workflows
Where is the main entry point of this application? Walk me through what happens when a user opens Excalidraw in their browser.
  1. excalidraw-app/index.html — an inline script immediately applies the saved theme (no flash), then loads index.tsx.
  2. excalidraw-app/index.tsx — registers the service worker (PWA), mounts <ExcalidrawApp /> into #root.
  3. excalidraw-app/App.tsx — loads saved drawing from localStorage, checks the URL for a collaboration room, fetches room data from Firebase if needed, then renders the core <Excalidraw /> component from packages/excalidraw.
What are the most important files in this project?

Top 10 for new contributors:

  1. packages/excalidraw/components/App.tsx — ~12,500 lines, handles all user interactions on the canvas.
  2. packages/element/src/types.ts — defines all element types (shapes, arrows, text, images).
  3. packages/excalidraw/types.ts — defines AppState and the public API surface.
  4. packages/excalidraw/appState.ts — default values for all editor state.
  5. packages/element/src/Scene.ts — single source of truth for all elements on the canvas.
  6. packages/element/src/newElement.ts — factory functions for creating elements.
  7. packages/excalidraw/actions/manager.tsx — centralized action system used for every operation.
  8. packages/element/src/renderElement.ts — draws elements onto the canvas using Rough.js.
  9. packages/excalidraw/history.ts — delta-based undo/redo system.
  10. excalidraw-app/collab/Collab.tsx — real-time collaboration via WebSocket + Firebase.
How does Excalidraw handle drawing shapes? Explain the flow from when a user clicks on the canvas to when a shape appears.
  1. Tool selected — app records the active tool (e.g. rectangle).
  2. Pointer down — screen coords converted to scene coords; a new element object is created (width=0, height=0) and immediately added to the scene.
  3. Drag — element's width/height updated many times per second; snapping is calculated; canvas redraws using Rough.js (which adds the hand-drawn wobbly style).
  4. Pointer up — element is finalized, committed to history, and broadcast to collaborators if in a shared session.
Can Excalidraw pick up an image directly from the internet or browser, or does it need to be saved locally?

No local save required for most methods. Supported ways to insert images:

  • Paste a copied image (Ctrl+V) — read directly from clipboard.
  • Paste an image URL — Excalidraw fetches and downloads it automatically.
  • Drag from a web page — browser passes the image data directly.
  • Image tool / drag from desktop — requires a local file.

All images are converted to base64 data URLs and stored in localStorage; no external URL links are kept.

Does Excalidraw have any integrated AI? If so, what is it?

Yes, two AI features:

  1. Text to Diagram (TTD) — a chat interface where you describe a diagram in plain English; AI responds with Mermaid syntax which is converted to Excalidraw shapes. Includes conversation history, auto-repair of broken diagrams, and rate limiting. Labeled "AI Beta".
  2. Magic Frame (Wireframe to Code) — draw shapes inside a special Magic Frame, click the wand button, and AI generates HTML from your wireframe. The actual AI model is not bundled — it's plugged in via a callback (onTextSubmit / generate), so embedders can use any model. Excalidraw.com uses its own Excalidraw+ backend.

A third tab offers Mermaid to Excalidraw (manual Mermaid editor, no AI).

How does the real-time collaboration system work? What happens when two users share a session?

Collaboration is built on two layers working together:

  1. Room creation — when a user clicks "Live collaboration", the app generates a unique room ID and a random encryption key. Both are embedded in the shareable URL — the server never sees the key.
  2. WebSocket sync — all participants connect to a Firebase-backed WebSocket server (excalidraw-app/collab/Collab.tsx). Every canvas change (element added, moved, deleted) is serialized and broadcast to all room members in real time.
  3. End-to-end encryption — data is encrypted in the browser before being sent, using the key from the URL. Even the server operator cannot read the drawing contents.
  4. Reconciliation — when a late-joining user enters the room, they receive the current scene state from the server. Conflicts are resolved by last-write-wins on a per-element basis.

There is no account system — access to a session is controlled entirely by possessing the shareable URL. Closing the tab ends your participation; the room persists as long as at least one user remains connected.

Summary

Excalidraw is a well-architected open-source whiteboard built as a Yarn monorepo, with a clean separation between the reusable core library (packages/excalidraw) and the full web application (excalidraw-app). The drawing engine works by creating element objects on pointer-down and continuously mutating them during drag, rendering everything through Rough.js to produce the signature hand-drawn aesthetic.

Images can be inserted directly from the browser or internet via clipboard and drag-and-drop without any local file required, and the app persists all data — including images as base64 — entirely in localStorage with no account needed.

The project includes two AI features (Text to Diagram chat and Magic Frame wireframe-to-code), both designed as pluggable interfaces so any LLM backend can be connected. Overall, Excalidraw prioritizes simplicity and openness: no login, no forced cloud storage, real-time collaboration via encrypted shareable links, and a public npm package that anyone can embed.
